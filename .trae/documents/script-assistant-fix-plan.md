# 剧情助手 Bug 修复与功能优化计划

## 问题清单

### 1. 双击菜单显示区域错误

* [ ] 双击节点后菜单没有出现在鼠标位置

* [ ] 需要获取鼠标点击坐标并传递给菜单组件

### 2. 分镜菜单干扰

* [ ] 剧情节点双击后显示分镜助手菜单项（不应出现）

* [ ] 需要根据项目类型过滤菜单项

* [ ] 剧情助手的菜单名称没有正常显示

### 3. 连线自动打开菜单

* [ ] 从节点拉出连线后会自动打开菜单和分镜助手一样的交互逻辑

### 4. 新建剧本交互流程重构

* [ ] 首次进入剧本项目时显示欢迎对话框

* [ ] 提供两个选项：

  * 导入剧本 + LLM 分析（已有资产）

  * 创建故事（用户输入概要）→ LLM 生成大纲

* [ ] LLM 生成后自动在画布上创建章节节点

* [ ] 初始只生成大纲（大章节标题和摘要）

* [ ] 支持逐步扩充完善内容

### 5. 剧本资产管理优化

* [ ] 资产面板默认收起（剧情内容为空白时）

* [ ] 添加"一键提取"按钮

* [ ] 一键提取功能：扫描画布节点内容，自动生成资产

  * 分析章节内容提取角色

  * 分析场景标题提取地点

  * 分析内容提取道具

* [ ] 导入完整剧本时直接生成资产

## 实现步骤

### 第一阶段：修复菜单位置与显示问题

#### 1.1 修复双击菜单位置

* [ ] 修改 `NodeActionToolbar` 或相关组件

* [ ] 获取双击位置坐标

* [ ] 将位置传递给菜单

#### 1.2 修复菜单项过滤

* [ ] 在菜单组件中检测项目类型 (`projectType`)

* [ ] 根据类型过滤菜单项

* [ ] 移除分镜相关菜单（AI图片、分镜生成等）

#### 1.3 修复菜单名称显示

* [ ] 检查菜单配置文件

* [ ] 添加剧情助手菜单项

#### 1.4 修复连线自动打开菜单

* [ ] 在连线处理逻辑中添加项目类型判断

* [ ] 剧本模式下禁止自动打开菜单

### 第二阶段：重构新建剧本流程

#### 2.1 创建欢迎对话框

* [ ] 新建 `ScriptWelcomeDialog.tsx`

* [ ] 两个选项：

  * 导入剧本：文件选择器 → 解析 → LLM分析 → 生成资产 + 章节

  * 创建故事：文本框输入概要 → LLM生成大纲 → 生成章节（无资产）

* [ ] 支持拖拽文件

#### 2.2 LLM 生成大纲

* [ ] 在 `scriptAnalyzer.ts` 添加 `generateOutline` 函数

* [ ] 生成大章节结构（仅标题 + 摘要）

* [ ] 在画布上创建章节节点

#### 2.3 一键提取生成资产

* [ ] 新建 `extractAssetsFromChapters` 函数

* [ ] 扫描所有章节节点内容

* [ ] 使用 LLM 或规则提取：

  * 角色（检测角色名、对白）

  * 地点（检测场景标题）

  * 道具（检测物品描述）

* [ ] 自动创建对应节点

#### 2.4 资产面板优化

* [ ] 默认收起状态（根据是否有资产动态判断）

* [ ] 添加展开/收起按钮

### 第三阶段：集成测试

#### 3.1 入口集成

* [ ] 在 `Canvas.tsx` 检测首次进入剧本项目

* [ ] 显示欢迎对话框

* [ ] 处理用户选择
