- [ ] 支持直接编辑场景地点信息

- [ ] 添加资产创建功能

### 2. 导出功能

#### 2.1 导出格式支持
- [ ] **TXT** - 标准剧本格式（纯文本）
- [ ] **Word (.docx)** - 微软 Word 格式（需要安装 docx 库）
- [ ] **JSON** - 结构化数据
- [ ] **Markdown** - Markdown 格式

#### 2.2 分支选择功能
- [ ] 检测剧本中的分支节点
- [ ] 创建分支选择对话框组件 (BranchSelectionDialog)
- [ ] 支持选择导出：
  - 主分支 (Main Branch)
  - 所有分支 (All Branches) - 合并导出
  - 指定分支 (Specific Branch)
- [ ] 分支可视化标记（区分主支/分支）

#### 2.3 导出内容结构（按剧本流程顺序）
```
1. 剧本基本信息
   - 标题
   - 类型/Genre
   - 总章节数
   - 分支信息（如果有）

2. 故事摘要/大纲
   - 按章节顺序排列的摘要

3. 角色档案
   - 角色名
   - 描述
   - 性格特点
   - 外貌特征

4. 场景地点清单
   - 场景名称
   - 描述

5. 关键道具清单
   - 道具名称
   - 描述

6. 章节详细内容（按顺序）
   - 章节标题
   - 场景标题 (Scene Heading)
   - 内容正文
   - 对话/动作
   - 分支标记（如有）

7. 埋点追踪
   - 伏笔点
   - 响应点
```

- [ ] 创建 scriptExporter.ts 服务

- [ ] 使用 docx 库生成 Word 文档

#### 2.5 UI 集成

## 分支检测逻辑

```typescript
interface BranchInfo {
  id: string;
  name: string;
  startChapter: number;
  endChapter: number;
  path: string[]; // 节点ID路径
  isMainBranch: boolean;
}

// 检测流程：
// 1. 从根节点开始深度优先遍历
// 2. 识别分支节点（isBranchPoint = true）
// 3. 构建分支树
// 4. 确定主分支（通常是最长或第一个分支）
```

## 导出格式详细设计

### TXT 格式（标准剧本格式）
```
================================================================================
                              剧本标题
                              类型: XXX
                              分支: 主分支 (或: 主分支 + X 个分支)
================================================================================

【故事大纲】
第1章: 摘要...
第2章: 摘要...
[分支A] 第3章: 摘要...
[分支A] 第4章: 摘要...
[分支B] 第3章: 摘要...
...

【角色】
角色名: 描述
      性格: ...
      外貌: ...

【场景】
场景名: 描述

【道具】
道具名: 描述

================================================================================
                                正文
================================================================================

第1章 章节标题
──────────────────────────────────────

INT. 场景 - 日

角色A
(动作) 对白内容...

[分支A] 第3章 章节标题
...
```

### Word 格式
- 使用 docx 库生成
- 包含标题页、目录、各章节内容
- 保留格式（加粗、斜体、对齐）
- 可编辑的 .docx 文件
- 分支内容用不同颜色或章节分隔

### JSON 格式
```json
{
  "title": "剧本标题",
  "genre": "类型",
  "branches": [
    {
      "name": "主分支",
      "isMain": true,
      "chapters": [...]
    },
    {
      "name": "分支 A",
      "isMain": false,
      "chapters": [...]
    }
  ],
  "characters": [...],
  "locations": [...],
  "items": [...]
}
```

### Markdown 格式
- 适合发布到博客或文档系统
- 清晰的分级标题结构
- 分支用子章节或标注区分
