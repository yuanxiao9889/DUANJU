# 剧本创作功能开发计划 (Phase 1)

## 一、概述

本计划基于 Storyboard-Copilot 现有框架，开发剧本创作功能。该功能利用 LLM 帮助用户生成剧本，剧本以章节节点形式分布在画布上。

**现有架构基础**:
- 节点画布系统 (@xyflow/react)
- AI 模型供应商适配 (fal/grsai/kie/ppio)
- 工具插件体系 (crop/annotate/split)
- 持久化层 (SQLite + Rust)

## 三、需求分析

### 3.1 核心功能模块

#### 模块一: 剧本导入与深度解构
- **文档上传**: 支持 PDF/Word/TXT 格式
- **智能章节拆分**: 识别章节标题，切分为独立节点
- **剧情摘要生成**: 50-100 字/节点，包含起因/冲突/结果/情感转向
- **全局实体提取**: 角色表、场景表、关键道具
- **逻辑埋点识别**: Setup/Payoff 伏笔响应识别
- **风格画像分析**: 对白占比、动作描写、语言风格

> **重要**: 以上 AI 生成内容需在**左侧资产面板**中集中展示，每栏可展开/收起，用户可编辑

#### 模块二: 新建剧本与骨架搭建
- **预设参数配置**: 类型(悬疑/爱情)、体量(电影/剧集)、章节数、核心冲突
- **结构模板注入**: 三幕式/救猫咪/英雄之旅
- **节点意图填充**: 用户输入简略想法，AI 补完逻辑链
- **分支可能性生成**: A/B/C 三种变体
- **世界观约束录入**: 全局禁令常驻约束

#### 模块三: 全局资产池 (剧本百科)
> **核心 UI**: 左侧资产面板，每栏可展开/收起，用户可编辑

- **剧情摘要**: 每个章节的摘要卡片（起因/冲突/结果/情感转向）
- **角色档案**: 角色列表，含性格/外貌/状态/关系
- **场景列表**: 场景地点，含描述/出现章节
- **关键道具**: 道具列表，含描述/关联章节
- **埋点追踪**: Setup/Payoff 伏笔响应对
- **风格画像**: 对白占比/动作详略/黑话术语

### 4.6 节点数据结构

```typescript
// 剧本根节点数据
interface ScriptRootNodeData {
  title: string;                    // 剧本名称
  genre: string;                   // 类型
  totalChapters: number;           // 总章节数
  styleProfile?: StyleProfile;     // 风格画像
}

// 剧本章节节点数据
interface ScriptChapterNodeData {
  chapterNumber: number;            // 章节序号
  title: string;                   // 章节标题
  content: string;                 // 正文内容 (含表格/格式)
  summary: string;                 // 剧情摘要 (50-100字)
  sceneHeadings: string[];         // 场景标题列表
  characters: string[];            // 登场角色
  locations: string[];             // 涉及场景
  emotionalShift: string;           // 情感转向
  setupRef?: string;              // 伏笔引用 (Setup)
  payoffRef?: string;             // 响应引用 (Payoff)
  isBranchPoint: boolean;         // 是否分支点
  branchType?: 'main' | 'branch'; // 连线类型
  linkedChapterId?: string;       // 链接的下一章节
  branches?: BranchOption[];       // 分支选项
}

// 分支选项
interface BranchOption {
  id: string;
  label: string;
  targetChapterId: string;
  description: string;
}

// 风格画像
interface StyleProfile {
  dialogueRatio: number;        // 对白占比
  actionDetailLevel: string;    // 动作描写详略
  slangTerms: string[];        // 黑话/术语
}

// 表格数据
interface ScriptTable {
  id: string;
  type: 'dialogue' | 'scene' | 'custom';
  columns: string[];
  rows: Record<string, string>[];
}
```
