# Qwen3 TTS 扩展手测清单

## 目标

验证完整扩展包目录是否能在应用里被加载、启用，并在画布中完成一次真实语音生成。

## 测试目录

直接加载这个目录：

- `E:\Storyboard-Copilot\extension-packages\qwen3-tts-complete`

不要加载这个模板目录：

- `E:\Storyboard-Copilot\extension-packages\qwen3-tts-complete-template`

## 测试步骤

1. 打开软件。
2. 点击标题栏里的“扩展中心”。
3. 选择“加载文件夹”。
4. 选择 `E:\Storyboard-Copilot\extension-packages\qwen3-tts-complete`。
5. 确认列表里出现 `Qwen3 TTS Complete`。
6. 点击“启用扩展”。
7. 观察进度条是否依次经过校验、运行时检查、模型检查。
8. 启用完成后，关闭扩展中心。
9. 在画布空白处打开节点菜单。
10. 确认能看到“配音文本”和“声音设计”两个节点。
11. 新建一个“配音文本”节点，输入一段中文测试文案。
12. 新建一个“声音设计”节点。
13. 将“配音文本”连接到“声音设计”。
14. 在“声音设计”里填写声音设定，例如“年轻女性，自然、清晰、平稳”。
15. 点击“生成语音”。
16. 等待完成后，确认画布中自动生成一个新的音频节点。
17. 点击播放，确认音频可播放。

## 成功标准

- 扩展包目录能被正确识别。
- 启用扩展不会报错。
- 画布节点菜单能出现 TTS 相关节点。
- 声音设计节点能真实生成音频，而不是 mock 结果。
- 自动生成的音频节点可以播放。

## 失败时优先检查

- 扩展中心里是否加载了错误目录。
- 是否误加载了 `qwen3-tts-complete-template`。
- 显卡和本地 CUDA 环境是否可正常使用。
- `runtime/python` 和 `runtime/models` 是否都存在于完整包目录里。
- 扩展中心错误提示里是否显示了缺失模型或运行时检查失败。
